# Codex Project Instructions

本项目的后续实质变更默认遵循以下约定：

- 涉及交易逻辑、风控、交易所对账、运行态、部署或线上风险处理的变更，必须在 `docs/ops/` 下新增或更新中文变更记录。
- 变更记录应包含背景、现场现象、根因、代码改造、验证结果、线上状态和后续注意事项。
- 提交信息使用中文，简洁描述主题。
- 提交前运行项目测试：`.venv/bin/python -m pytest`。
- 默认将代码变更和对应变更记录放在同一个 commit 中，便于后续回溯。
- 用户明确要求实施并提交/推送时，完成测试后推送到当前分支对应远端。
- 不提交 `.env`、数据库、日志、密钥、交易所原始敏感响应或其他运行时私密文件。

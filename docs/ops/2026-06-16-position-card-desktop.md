# 2026-06-16 当前持仓桌面卡片化

## 背景

持仓页移动端已经使用卡片展示当前持仓，但桌面端仍使用横向表格。

当前持仓字段较多，且包含保护单状态、实时价格来源、手动操作按钮。桌面表格在字段增加后可读性下降，
操作区也容易拥挤。

## 改动

- 当前持仓区域统一改为卡片网格，桌面和移动端复用同一套展示结构。
- 桌面端自适应多列展示，窄屏自动退化为单列。
- 保留当前持仓所有字段：
  - 币种、方向、杠杆、开仓时间
  - 未实现盈亏、保证金收益率
  - 持仓数量、保证金、开仓价、标记价/最新价、名义价值、强平价格
  - 止损保护、止盈保护
  - 补止盈止损、接管保护、启用并接管、手动平仓
- 多仓/空仓继续用左侧绿色/红色边框区分。
- 缺保护或币种已禁用继续使用黄色警示边框。
- 普通挂单和条件单保持原来的桌面表格 + 移动卡片模式。

## 影响

该改动只影响前端展示层：

- 不改变后端 API。
- 不改变 WebSocket 数据链路。
- 不改变 engine、风控、下单、补单和平仓逻辑。

## 验证

- `npm --prefix web/frontend run build`：通过。
- 桌面当前持仓以卡片网格展示。
- 移动端当前持仓仍为单列卡片。
- 普通挂单和条件单桌面表格不受影响。

# 2026-06-16 持仓页实时行情叠加

## 背景

持仓页原来显示 `本地快照 同步于 xxxx`，容易误解为页面数据完全静态。

实际数据链路分为两类：

- 账户持仓：来自后端账户投影，主要由 Binance 私有流事件驱动，REST 账户快照补全。
- 行情价格：来自公开行情 WebSocket。

私有流可以快速同步持仓数量、开仓价和未实现盈亏等账户变化，但不保证包含强平价格、
完整保证金等字段；这些字段仍需要 REST 账户快照补全。

## 改动

### 数据源标签

持仓页标签按后端 `positions_source` 明确展示：

- `exchange`：交易所实时
- `account_projection`：账户投影实时
- `db_snapshot`：本地快照

页面顶部拆分两个时间：

- `账户同步于`：后端账户投影或 REST 快照更新时间。
- `行情更新于`：前端公开行情 WebSocket 最近 ticker 时间。

### 前端实时行情叠加

持仓页会按当前持仓币种连接环境化行情流：

- testnet 页面：`/ws/testnet/market`
- mainnet 页面：`/ws/mainnet/market`

收到 ticker 后，前端用实时标记价覆盖展示字段：

- `mark_price`
- `notional`
- `unrealized_pnl`
- `roi_pct`

标记价旁展示 `实时` 标签，表示该行价格和盈亏已用最新行情覆盖。

### 连接生命周期

- 当前无持仓时不建立行情连接。
- 持仓币种变化时自动增删对应行情连接。
- 页面切换 testnet/mainnet 时关闭旧连接并清空旧环境行情。
- 页面进入后台时关闭行情连接，回到前台后重新连接。
- 页面卸载时移除事件监听并关闭连接，避免重复订阅。

## 边界

强平价格不做前端估算，仍展示后端账户快照中的交易所字段。

原因：

- 强平价依赖风险限额、保证金模式、维持保证金、账户余额等交易所内部规则。
- 前端仅凭 ticker 和当前行字段估算容易给出错误安全边界。
- REST 账户快照返回的 `liquidationPrice` 是更可信来源。

因此强平价的实时性等于后端 REST 账户快照刷新频率，不等于行情 ticker 频率。

## 验证

- 前端持仓页桌面表格和移动卡片均使用实时叠加后的持仓行。
- 行情 WebSocket 路由沿用 K 线页已有 `wsPath('/market')` 环境切换机制。
- `npm --prefix web/frontend run build`：通过。

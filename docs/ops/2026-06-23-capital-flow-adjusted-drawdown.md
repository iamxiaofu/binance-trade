# 2026-06-23 资金流调整后的回撤熔断

## 背景

原回撤熔断直接比较 Binance 原始账户权益与当日权益峰值。用户从 USDⓈ-M 合约账户
转出 USDT 时，账户权益会下降，但这不是交易亏损。如果不识别资金流，正常转出可能被
误判为达到 `max_drawdown_pct`，继而暂停策略并触发自动平仓。

Binance 提供两类可交叉核对的数据：

- 私有流 `ACCOUNT_UPDATE`：`a.m` 表示余额变动原因，`B[].bc` 表示不含盈亏和手续费
  的余额变化；
- REST `/fapi/v1/income`：资金流水包含 `incomeType`、`income`、`time` 和同类型内
  唯一的 `tranId`。

## 风控口径

系统同时保留以下指标：

- `raw equity`：交易所原始账户权益，用于保证金、下单额度和账户历史曲线；
- `day net capital flow`：本地自然日内外部净资金流，转入为正、转出为负；
- `risk equity = raw equity - day net capital flow`；
- `risk day drawdown`：`risk equity` 相对当日最高 `risk equity` 的回撤。

示例：日初权益 1000 USDT，转出 300 后原始权益为 700，净资金流为 -300，
风控权益仍为 `700 - (-300) = 1000`，因此不会产生交易回撤。

转入也从风控权益中扣除，避免充值虚增当日峰值和后续亏损额度。

日亏熔断的百分比基数同样改用风控权益，避免转出后错误缩小日亏额度、转入后错误放大
日亏额度。

## 资金流水账本

新增 `capital_flows` 表：

- 私有流先写 `provisional`，唯一键使用 event key；
- REST income 写 `confirmed`，唯一键使用 `incomeType + tranId + asset`；
- REST 到达后按资产、金额和时间窗口匹配私有流，将临时记录标记为 `matched`；
- 当日净资金流只汇总 `confirmed` 与尚未匹配的 `provisional`，避免双计；
- 重启后从持久化账本和 REST income 恢复，不依赖进程内存。

REST 按 1000 条分页拉取当日流水；20 页仍未结束时标记为 `INCOMPLETE`，不能宣称
资金流水已完整。

## 回撤核对保护态

首次达到回撤阈值时不立即平仓：

1. 进入 `CAPITAL_FLOW_RECONCILING`；
2. 暂停新开仓；
3. 不执行任何自动平仓；
4. 等待私有流和 REST income 完成归因；
5. 若确认存在转出且调整后回撤低于阈值，自动解除本次临时保护；
6. 若账本完整、30 秒核对窗口已过且调整后回撤仍超限，才触发正常
   `MAX_DRAWDOWN` 熔断。

若 income 查询失败或流水不完整，只维持核对暂停，不允许将未知余额下降当作真实亏损
执行紧急平仓。

已有人工暂停、重启保护或其他风控暂停不会被资金流核对逻辑自动解除。

## 私有流边界

`ACCOUNT_UPDATE` 不再直接更新熔断使用的账户权益。它只更新实时账户投影和临时资金
流水；风控权益必须在 REST 余额与 income 账本完成同一轮核对后更新，避免使用“新资金
流 + 旧余额”组合制造虚假峰值。

## 看板

新增并区分：

- 账户原始权益；
- 今日净资金流；
- 资金流调整后风控权益；
- 资金流核对状态；
- 当日风控回撤；
- 账户历史原始权益回撤。

熔断口径标识为 `FLOW_ADJUSTED_DAILY_EQUITY_HIGH_WATER_MARK`。

## 部署兼容

旧版本保存的当日峰值基于原始权益，不能直接与新风控权益比较。首次升级时通过
`risk.capital_flow.model_version=1` 识别迁移，并使用第一笔完成资金流水核对的有效余额
初始化新峰值，避免升级时误熔断。之后峰值、净资金流和核对状态均持久化恢复。

## 验证范围

- 单次和分批转出不产生交易回撤；
- 转入不抬高风控峰值；
- 转出与真实亏损并存时只计算真实亏损；
- 私有流与 REST 重复流水去重；
- 重启恢复；
- 首次真实超限先核对、核对期间不平仓；
- income 失败或超过分页完整性上限时保持暂停且不平仓；
- 核对确认真实亏损后正常触发仅限 Engine 托管数量的熔断平仓。
